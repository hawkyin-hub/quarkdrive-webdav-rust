## 1. P1-1 后端崩溃自愈（main.rs）
**问题**：`health.rs` 的 `repair()` 在 `WebDavUnreachable` 时调 `restart_signal.notify_waiters()`，但全代码库没有任何地方 `.notified().await`，后端/proxy 崩溃后无法自愈。

**改动**（仅全功能模式，serve_only 维持"退出由外部重启"语义）：
- `build_handler()` 后新增 `let handler_template = dav_handler.clone();` / `let fs_template = fs_for_webdav.clone();`（DavHandler 与 QuarkDriveFileSystem 均 Clone）。
- 全功能模式新增 `server_slots: Arc<std::sync::Mutex<Option<(AbortHandle, AbortHandle)>>>`，backend/proxy spawn 后注册各自的 `abort_handle()`。
- `health::spawn_loop` 之后新增 **watchdog task**：`loop { restart_signal.notified().await; abort 当前两个 handle; sleep 1s; 用 template 重建 WebDavServer + proxy::run 并 spawn; 注册新 abort handle; wait_for_port 校验; }`。
- `main` 末尾 shutdown 处增加 `server_slots` 内 handle 的 abort。

**回滚**：`cp .audit-backup-20260707/src/main.rs quarkdrive-webdav/src/main.rs`

## 2. P1-2 空文件上传（vfs.rs）
**问题**：0 字节 PUT 时 `write_buf` 不被调用 → `is_uploading=false` → `flush()` 提前 `return Ok(())`，空文件永不创建。`upload_mini_byte_file` 是为它设计但从未被调用且自身不完整。

**改动**：
- `struct QuarkDavFile` 新增字段 `write_mode: bool`。
- `QuarkDavFile::new(..)` 新增参数 `write_mode: bool`；`open()` 两处调用点传入 `options.write`。
- `flush()` 的 `if !self.upload_state.is_uploading` 分支改为：`if !self.write_mode { return Ok(()) }`（只读 open 的 no-op）；否则（写模式空文件）调 `self.prepare_for_upload().await?` 建立上传态，随后走 `do_flush`（size=0 → chunk_count=0 → 空 commit，已验证可处理）。

**回滚**：`cp .audit-backup-20260707/src/vfs.rs quarkdrive-webdav/src/vfs.rs`

## 3. P2-1 PROPFIND 缓存写失效（proxy.rs + vfs.rs）
**问题**：`proxy.rs` 的 `PROPFIND_CACHE`（5s TTL）在 PUT/DELETE/MKCOL/MOVE 成功后不失效，导致新文件最长 5s 不可见 / 删掉的文件还在。

**改动**：
- `proxy.rs` 新增 `pub fn invalidate_propfind(path: &str)`：锁全局 cache，清掉 `path`（任意 depth）及其子路径前缀的条目；`path=="/"` 时 `clear()`。
- `vfs.rs` `impl QuarkDriveFileSystem` 新增 `pub fn invalidate_proxy_propfind(&self, dav_path: &Path)` 转调。
- 在所有"写成功后失效 dir_cache"的位置同步调 `invalidate_proxy_propfind`：`after_flush`、spawn 上传任务 `early_finish` 与正常成功路径、`create_dir`、`remove_dir`、`remove_file`、`rename`（from/to 及其父）。

**回滚**：分别 `cp .audit-backup-20260707/src/proxy.rs` 与 `src/vfs.rs`。

## 4. P2-2 覆盖写删旧时机（vfs.rs）
**问题**：`do_flush` 在 `up_pre` **之前** `remove_file` 旧 fid；若 `up_pre`/`up_hash` 失败，旧文件已删、新文件未建 → 数据丢失。

**改动**：
- `do_flush` 开头新增 `let old_fid = if !self.file.fid.is_empty() { Some(self.file.fid.clone()) } else { None };`（在 `up_pre` 覆盖 `self.file.fid` 之前捕获）。
- 删除"先删旧文件"那段（改为注释说明延后）。
- 在三个成功路径删除旧 fid（均带 `old != new_fid` 且 `!new_fid.is_empty()` 守卫，避免误删新文件）：
  - `up_pre` 秒传成功；`up_hash` 秒传成功；spawn 任务 `commit+finish` 成功后（通过 `old_fid_opt`/`no_trash` 传入闭包，用 `drive_inner.remove_file(old, !no_trash)`）。
- MD5 相同 / 同 size 跳过路径不变（不删旧，内容未变）。

**回滚**：`cp .audit-backup-20260707/src/vfs.rs quarkdrive-webdav/src/vfs.rs`

## 5. P2-3 parent_path unwrap 兜底（vfs.rs）
**问题**：`self.file.parent_path.as_ref().unwrap()` 5 处，`parent_path` 为 None 时 panic。

**改动**：新增 `fn parent_path_str(&self) -> String`（`parent_path.clone().unwrap_or_else(|| parent_dir.to_string_lossy().into_owned())`），替换全部 5 处 unwrap（3 处 `register_active_write`、`do_flush` spawn 的 `parent_path`、`after_flush` 的 `parent_path`）。

**回滚**：`cp .audit-backup-20260707/src/vfs.rs quarkdrive-webdav/src/vfs.rs`

## 6. P2-4 dfs 分页重试（cache.rs）
**问题**：`dfs()` 并发分页 `Err(e) => break` 丢弃后续所有页 → 偶发 5xx/限流时目录残缺并被 moka 缓存。

**改动**：每页 fetch 改为最多 3 次（400ms×attempt 退避）；`Err` 分支由 `break` 改为 `warn!` + 继续后续页（单页持续失败只丢该页，不再丢全部）。

**回滚**：`cp .audit-backup-20260707/src/cache.rs quarkdrive-webdav/src/cache.rs`

## 7. P4-1 serve_only 挂载点（main.rs）
**问题**：`serve_only` 分支整体忽略 `--mount-point`，硬编码 `/Volumes/LocalQuark`。

**改动**：
- `mount_supervisor_loop` 签名增加 `mount_point: String`，函数体内 `const MOUNT_POINT` 改为 `let mount_point_str = mount_point.as_str();`（其余引用同步改名）。
- `serve_only` 分支：`let serve_mount_point = expand_home(&opt.mount_point);`，传入 supervisor，并用 `serve_mount_point.clone()` 取代 `PathBuf::from("/Volumes/LocalQuark")` 供 admin/health/mount_cfg 使用。
